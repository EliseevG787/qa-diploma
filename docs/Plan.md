## План автоматизации тестирования комплексного сервиса, взаимодействующего с СУБД и API Банка.
### Перечень автоматизируемых сценариев:
Предусловие:

Пользователь находится на странице оплаты.

Нажата кнопка «Купить» или «Купить в кредит».

Корректные входные данные:
1. Набор карт для тестирования:
   
     4444 4444 4444 4441 со статусом "APPROVED";
   
     4444 4444 4444 4442 со статусом "DECLINED".
2. Месяц: 01-12, 2 цифры;
3. Год: последние 2 цифры года;
4. Корректная дата – в течение 5 лет, начиная от текущей.
4. Владелец: имя на кириллице;
5. CVC/CVV: 3 цифры.

Тестируется поддержка двух СУБД:
*	MySQL
*	PostgreSQL


#### Проверить возможность оплаты тура картой со статусом "APPROVED".
Шаги:
1. Заполнить форму оплаты картой (поля: Номер карты, Месяц, Год, Владелец, CVC/CVV) корректными данными.
2. Нажать кнопку «Продолжить».

Ожидаемый результат:

Появляется сообщение: «Операция одобрена банком». В базе данных появилась запись APPROVED, сумма трансакции соответствует заявленной.


#### Проверить оплату картой со статусом "DECLINED".
Шаги:
1. Заполнить форму оплаты картой (поля: Месяц, Год, Владелец, CVC/CVV) корректными данными.
2. В поле «Номер карты» ввести номер карты со статусом "DECLINED".
3. Нажать кнопку «Продолжить».

Ожидаемый результат:

Появляется сообщение: «Ошибка! Банк отказал в проведении операции». В базе данных появилась запись "DECLINED".

#### Перечень используемых инструментов:
*	Java 11 - язык программирования для написания кода автотестов;
*	IntelliJ IDEA - интегрированная среда разработки программного обеспечения, поддерживающая инструменты тестирования;
*	Gradle - система автоматической сборки;
*	JUnit5 - библиотека для модульного тестирования программного обеспечения на языке Java;
*	Lombok для упрощения кодирования;
*	Docker —автоматизирует разворачивание контейнеров MySQL, PostgreSQL, Node.js
*	Selenide - выводить подробную информацию о веб-элементах, заполнять и отправлять форму;
*	Faker - библиотека для генерирования тестовых данных (Номер карты, Месяц, Год, Владелец, CVC/CVV);
*	Allure для более наглядных отчетов.


#### Перечень и описание возможных рисков при автоматизации:
*	Недоступность сайта или страницы;
*	Проблема полного покрытия кода автотестами;
*	Необходимость модифицировать автотесты при частом изменении приложения;
*	Увеличение стоимости автоматизации тестирования;
*	Отсутствие тестовых данных;
*	Отсутствие доступа к реальной базе данных.


#### Перечень необходимых специалистов для автоматизации:
*	Специалист по автоматизированному тестированию на Java;


#### Интервальная оценка с учётом рисков (в часах):
Необходимое время на тестирование с учетом рисков - 55 часов (5 часов — подготовка инфраструктуры, 40 часов — написание автотестов, 10 часов — отчётность).
